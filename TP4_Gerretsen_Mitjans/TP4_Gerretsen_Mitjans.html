The reason it is so famous in machine learning and statistics communities is because the data requires very little preprocessing (i.e. no missing values, all features are floating numbers, etc.).

Observation de l'arbre

Les lignes de chaque noeud dcrivent, dans l'ordre :

  • le critre de dcision (inexistant si le noeud est une feuille)
  • Gini est un critre qui mesure l'impuret du noeud (vaut 0 pour les feuilles), ce critre sert de critre d'arrt
  • Samples est la taille du sous-ensemble correspondant au noeud
  • Value est la rpartition des classes du sous-ensemble
  • class est la classe majoritaire ; pour les feuilles, c'est la fonction de dcision

A la premire division, tous les seposa (classe 0) sont distingus du reste. C'est ce que l'on a trouv qualitativement dans les prliminaires, la vue du graphe, puisque cette classe tait linairement sparable, par projection sur une entre.

Cet arbre a cependant un inconvnient majeur, puisque 3 niveaux de profondeur sont ddis uniquement classer 7 individus, ce qui rajoute de la complexit l'arbre, et nuit donc sa robustesse. Nous avons besoin d'un jardinier rgularisant afin d'laguer notre arbre.

2 approches sont possibles : projeter simplement les donnes sur 2 des 4 entres, ou bien faire une PCA pour extraire les 2 premires composantes principales sur lesquelles projeter les donnes.

Visualisation avec PCA

Nous allons faire une PCA des donnes afin de projeter les donnes et les dcisions dans un espace 2D visualisable. Voici les tapes suivre :

  • PCA 2 composantes des donnes (fournit en particulier une matrice de projection $\Sigma V^*$ de $R^4 \rightarrow R^2$ ; les 2 premires composantes principales $v_1$ et $v_2$ sont dans la matrice $V^*$).
  • Faire la projection des donnes 4D pour en visualiser l'aspect en 2D. En particulier, rcuprer les bornes min et max dans lesquelles elles vivent dans $Vect(v_1,v_2)$.
  • Crer une grille 2D entre ces bornes.
  • Faire la projection inverse de la grille, on obtient un ensemble de points contenus dans un plan 2D d'un espace 4D.
  • Lancer la prdiction sur cet ensemble obtenu. On obtient une prdiction pour chaque point.
  • Colorer les points de la grille 2D selon ces prdictions. On visualise alors une projection de la frontire de dcision sur $Vect(v_1,v_2)$.

Feature selection using random forests byproducts¶

One of the interesting use cases for random forest is feature selection. One of the byproducts of trying lots of decision tree variations is that you can examine which variables are working best/worst in each tree.

When a certain tree uses one variable and another doesn't, you can compare the value lost or gained from the inclusion/exclusion of that variable. The good random forest implementations are going to do that for you, so all you need to do is know which method or variable to look at.

View feature importance¶

While we don't get regression coefficients like with ordinary least squares (OLS), we do get a score telling us how important each feature was in classifying. This is one of the most powerful parts of random forests, because we can clearly see that petal width was more important in classification than sepal width.

Etude des paramtres

  • max_depth : impose une profondeur maximale aux arbres, sans maximum par dfaut
  • min_samples_leaf : quand on arrive ce nombre minimum d'chantillon, le noeud devient une feuille et on arrte la division, 1 par dfaut

Step 5: a small example of regression using random forests¶

Random forest is capable of learning without carefully crafted data transformations. Take the the $f(x) = \sin(x)$ function for example.

Based on this example, we will illustrate how the random forest isn't bound by linear constraints.

# Pruning function (useful ?)
def prune(decisiontree, min_samples_leaf = 1):
    if decisiontree.min_samples_leaf >= min_samples_leaf:
        raise Exception('Tree already more pruned')
    else:
        decisiontree.min_samples_leaf = min_samples_leaf
        tree = decisiontree.tree_
        for i in range(tree.node_count):
            n_samples = tree.n_node_samples[i]
            if n_samples <= min_samples_leaf:
                tree.children_left[i]=-1
                tree.children_right[i]=-1
